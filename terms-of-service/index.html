3. Description of Service

PaperTrail Academic provides tools for writing monitoring, analysis, and authorship consistency review. The Service includes:

  • PaperTrail Inspect: Revision history analysis, writing session reconstruction, and behavioral pattern detection based on Google Docs revision data. Core Inspect features are available to all users without an account. Paste analysis and struggle moment detection require an active StyleMatch subscription.
  • StyleMatch: Local, on-device stylometric comparison of two writing samples across eight metrics to assess authorship consistency. No text is transmitted when using StyleMatch. Users who create a free account receive three complimentary StyleMatch runs. Unlimited access requires an active subscription.
  • Verify: AI-assisted writing analysis that transmits submitted text to a third-party AI provider (Anthropic, Inc.) for processing. Available as a Consistency Report or a Baseline Deviation Report. See Section 8 for data handling details.
  • PaperTrail Write: A web application (write.papertrailacademic.com) for timed classroom writing sessions. Teachers create assignments, open sessions, and monitor student writing in real time. Students join via a code — no student account is required. Session data is stored transiently and purged automatically after the session ends. See Section 8 for data handling details.

8. Data, Privacy, and Student Text

StyleMatch (on-device): All stylometric computations are performed locally in your browser. No text you enter into the StyleMatch panel is transmitted to PaperTrail Academic or any third party. This feature is privacy-safe by design.

Verify (AI-assisted analysis): When you run a Verify report, the text you submit is:

  1. Transmitted to PaperTrail Academic's secure backend (Supabase, hosted on AWS infrastructure)
  2. Forwarded to Anthropic, Inc.'s API for AI-assisted analysis
  3. Returned as a structured report and saved to your account's report history

Text submitted through Verify is not stored by PaperTrail Academic beyond the metadata required to save your report (report ID, scores, timestamps, and any student name or assignment label you enter). The full essay text is not retained after the report is generated.

Anthropic processes submitted text under its commercial API terms, which prohibit using API inputs for model training. As of September 2025, Anthropic retains API inputs for 7 days before automatic deletion. See anthropic.com/privacy for their full policy.

PaperTrail Write (web application): PaperTrail Write is designed as a transit system, not a data vault. Student writing data is held transiently in a secure backend (Supabase, hosted on AWS) for the duration of the session.

The teacher is the data custodian. After a session ends, the teacher downloads a session report containing student writing and process logs. Student submission data is then purged automatically from PaperTrail's servers — untimed sessions after 30 days, timed sessions after 7 days, and ended sessions within 24 hours. These purges are automatic and cannot be reversed.

For PaperTrail Write specifically:

  • Students join sessions using a code — no student account, email address, or personal identifier is required beyond their name as entered by the student
  • Teachers may optionally connect a Google Drive account to attach source documents to assignments. Only the specific file selected by the teacher is accessed; PaperTrail Write does not browse or index other Drive content
  • Student writing data is never used for any purpose other than displaying it to the assigned teacher
  • Student writing data is never sold, shared with third parties, or used to train any AI model
